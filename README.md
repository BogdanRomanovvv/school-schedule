# School Schedule Generator

Полнофункциональное веб-приложение для автоматической генерации школьного расписания. Разработал как пет-проект, чтобы разобраться с Go, REST API и контейнеризацией.

## Стек

**Backend:** Go 1.21, chi router, sqlx, PostgreSQL 16  
**Frontend:** React 18, TypeScript, Vite, Axios  
**Инфраструктура:** Docker, Docker Compose  
**База данных:** PostgreSQL 16

## Что умеет

- Генерирует недельное расписание для 11 классов с учётом ограничений
- Исключает конфликты: один учитель не может быть в двух классах одновременно
- Алгоритм компактизации убирает «окна» в расписании (3 стратегии)
- REST API для управления классами, предметами, учителями, учебным планом
- Экспорт расписания в Excel

## Алгоритм генерации расписания

Задача — NP-сложная, реализовал жадный трёхфазный алгоритм:

1. **Phase 1 — назначение учителей:** каждому предмету класса назначается наименее загруженный доступный учитель. Классные руководители (1–4 кл.) принудительно назначаются в свой класс.

2. **Phase 2 — размещение уроков по слотам:** для каждого урока выбирается слот с учётом:
   - Контигуозного размещения (нет «дырок» с самого начала)
   - Предпочтения дней, где у учителя уже есть уроки
   - Ограничения max уроков в день на класс

3. **Phase 3 — компактизация (compact):** итеративно устраняет оставшиеся окна тремя стратегиями:
   - **Strategy A:** сдвигает урок из позднего слота в «дыру» (если учитель свободен)
   - **Strategy B:** переносит последний урок другого дня (без создания новой дыры)
   - **Strategy C:** меняет уроки двух классов местами через общего учителя

Результат: 0 конфликтов учителей, 0 дублирования предметов, минимум «окон».

## Запуск

**Требования:** Docker Desktop

```bash
git clone https://github.com/BogdanRomanovvv/school-schedule.git
cd school-schedule
docker-compose up --build
```

| Сервис    | URL                       |
|-----------|---------------------------|
| Frontend  | http://localhost:5173     |
| Backend   | http://localhost:8080     |
| DB        | localhost:5432            |

## API

После запуска доступны следующие эндпоинты:

```
POST   /api/schedule/generate    # Генерация расписания
GET    /api/schedule             # Получить расписание
GET    /api/schedule/export      # Экспорт в Excel
GET    /api/classes              # Список классов
GET    /api/teachers             # Список учителей
POST   /api/teachers             # Добавить учителя
PUT    /api/teachers/:id         # Обновить учителя
DELETE /api/teachers/:id         # Удалить учителя
GET    /api/subjects             # Список предметов
GET    /api/curriculum           # Учебный план
POST   /api/curriculum           # Добавить запись в план
```

## База данных

Схема инициализируется автоматически при первом запуске (`migrations/001_init.sql`).

```
classes → curriculum → schedule
subjects ↗         ↗
teachers ──────────┘
```
